## 1. 文档目的

本文定义一个真实可部署、可运营、可评测的企业级 RAG 知识平台。平台以文档管理、可验证的 AI 问答和知识图谱为核心，支持多格式文档与单页 URL 导入，通过向量检索、关键词检索、RRF 融合、Reranker 重排和 Neo4j 图谱推理实现多路召回；通过 LangGraph 构建只读 Agentic RAG；通过 Redis 和 Mem0 提供分层记忆；通过流式 ASR、SSE 与流式 TTS 提供半双工语音交互。

本文是产品、设计、研发、测试、部署和验收的共同基线。具体模型供应商、硬件型号和实施日历属于部署或实施计划配置，不改变本 PRD 的产品边界。

## 2. 产品背景与问题

企业知识通常散落在 PDF、Office 文件、表格、Markdown、JSON 和网页中，存在以下问题：

1. 文档格式复杂，扫描件、表格、幻灯片和旧版 Office 文件解析质量不稳定。
2. 纯向量检索容易遗漏专有名词、编号和精确短语，纯关键词检索又难以理解语义。
3. 传统 RAG 对关系型、多跳型问题能力不足，且容易在证据不足时生成看似合理的答案。
4. 文档、向量、关键词索引和图谱分散在多个存储中，更新、撤权和删除容易不一致。
5. AI 调用过程不可见，质量退化、错误引用、模型成本和用户反馈难以定位。
6. 企业用户要求权限隔离、引用溯源、审计、备份恢复和数据出网控制，不能停留在演示级能力。
7. 多轮对话既需要短期上下文，又需要可控的长期记忆，但不能把个人信息自动扩散为组织事实。

## 3. 产品愿景

让企业成员能够用自然语言或语音访问被授权的内部知识，并获得带原文引用、可验证、可追踪的答案；让管理员能够可靠导入和治理知识、审核图谱、评测质量、追踪故障并控制数据出网。

## 4. 产品目标

### 4.1 业务目标

- 将企业多格式资料统一沉淀为可浏览、可检索、可问答和可治理的知识空间。
- 普通用户无需理解向量、关键词或图谱参数，即可获得有证据的回答。
- 管理员能够观察文档从上传到可用的每个阶段，并能恢复失败任务。
- 复杂关系问题可通过图谱与文档证据联合推理。
- 通过评测集和线上反馈持续比较检索、Prompt、模型和策略版本。

### 4.2 产品目标

- 支持 PDF、DOCX、DOC、XLSX、XLS、PPTX、PPT、TXT、MD、CSV、JSON 和单页 URL。
- 支持扫描 PDF OCR，以及 DOCX/PPTX 等文档中关键图片的可选 OCR。
- 使用 pgvector 与 Elasticsearch 实现向量、BM25 双路召回。
- 使用 RRF 融合候选，使用 Reranker 做精排。
- 使用 Neo4j 管理带来源证据和审核状态的知识图谱。
- 使用 LangGraph 实现受限、只读、可追踪的 Agent 决策。
- 使用 Redis 管理滑动窗口、会话摘要、缓存和异步任务通道。
- 使用 Mem0 管理用户私有长期记忆和经审核的组织记忆。
- React 使用 Vercel AI SDK UI 管理消息状态、流式内容、工具状态、引用和取消。
- NestJS 通过兼容 AI SDK UI Message Stream 的 SSE 输出流式文本，使用独立 WebSocket 传输流式音频。
- 使用 LangGraph Studio 做本地调试，使用自托管 Langfuse 做生产 AI Trace 与评测。

### 4.3 非目标

首期不包括：

- 多租户 SaaS、计费套餐和商业化运营后台。
- OIDC、LDAP、AD 或企业 SSO。
- Kubernetes、高可用集群和自动扩缩容。
- 整站爬虫、登录态采集、定时网站同步。
- 公网搜索和未授权外部知识源。
- 可写数据库、邮件、HTTP 业务操作等通用执行 Agent。
- 全双工语音、插话打断、回声消除。
- 移动端原生 App。
- 第三方插件市场。

## 6. 用户与权限

### 6.1 管理员

管理员可以：

- 创建、禁用、重置本地用户账号。
- 创建部门和用户分组。
- 创建知识空间并设置空间成员权限。
- 上传、更新、停用、恢复和删除文档。
- 查看导入任务、失败原因和重试记录。
- 配置模型供应商、模型、限额和出网策略。
- 审核、合并、拆分、纠正和发布图谱实体与关系。
- 管理组织共享记忆。
- 创建评测集、执行评测、比较版本。
- 查看系统指标、AI Trace、操作审计和数据出网记录。
- 执行备份、恢复验证和数据重建操作。

### 6.2 普通成员

普通成员可以：

- 浏览被授权的知识空间和文档。
- 在被授权范围内搜索、文本问答和语音问答。
- 查看引用原文、页码、表格区域和文档预览。
- 创建、重命名、归档和删除自己的会话。
- 对回答点赞、点踩、标记引用错误或提交纠错说明。
- 查看、编辑、删除或关闭自己的长期记忆。
- 在获得空间级“编辑”或“管理”权限后管理对应空间内容。

### 6.3 权限模型

系统采用 RBAC 与资源 ACL 组合：

- 系统角色只区分管理员和普通成员。
- 系统管理员不受空间授权限制，并可以执行所有空间治理操作。
- `view`：浏览空间、搜索和问答、查看已发布图谱、打开引用、预览和下载有权文档、提交回答反馈。
- `edit`：包含 `view`，并可上传文件、导入 URL、创建新文档、上传新版本、编辑空间内任意文档的元数据，以及停用或恢复空间内任意文档；不能修改成员权限、文档级 ACL 或执行删除。
- `manage`：包含 `edit`，并可修改空间设置、管理空间成员、设置文档级 ACL、软删除和恢复任意空间文档、治理该空间图谱、管理该空间评测集。
- 文档默认继承知识空间 ACL。文档级 ACL 只能进一步收紧访问范围，不能向没有空间 `view` 权限的主体授予访问。
- 首期不提供显式 `deny` 规则。存在文档级 ACL 时，访问条件为“拥有空间 `view` 且用户、所属部门或用户组出现在文档允许列表中”；不存在文档级 ACL 时仅按空间权限判断。
- 所有检索路径必须在召回阶段过滤权限，不允许先召回再仅在界面隐藏。
- pgvector 查询、Elasticsearch 查询、Neo4j 证据回填、文档预览、下载和缓存均执行同一权限快照。
- 用户被撤权或文档被停用后，在线查询立即不可见，并异步清理相关缓存。
- 审计记录包含操作者、对象、动作、结果、时间、来源 IP 和关联请求 ID。

## 8. 核心用户流程

### 8.1 文件导入

1. 管理员或有空间编辑权限的成员选择知识空间。
2. 用户选择一个或多个受支持文件。
3. 前端执行基础格式和大小提示，服务端执行权威校验。
4. 系统保存原文件并创建文档与文档版本记录。
5. 安全服务执行文件签名、MIME、病毒、压缩炸弹、宏和嵌入对象检查。
6. Python 解析服务按格式提取结构化内容；扫描件和关键图片进入 OCR。
7. 系统规范化标题、段落、列表、页码、工作表、表格、幻灯片和备注。
8. 系统计算内容哈希，处理重复文件和新版本。
9. 系统进行结构感知切分，生成父子块和元数据。
10. 系统始终写入 Elasticsearch；空间允许 Embedding 时才生成向量并写入 pgvector，否则记录 `vector_index_status=SKIPPED_POLICY`。
11. Elasticsearch 成功，且按策略启用的向量索引也成功后，版本原子切换为“可检索”。
12. 空间启用图谱抽取时异步生成候选图谱并进入待审核区；未启用时跳过图谱任务。完成或跳过后文档进入“完全就绪”。

### 8.2 单页 URL 导入

1. 用户输入 HTTP 或 HTTPS URL。
2. 系统校验协议、DNS 解析结果、重定向链、目标 IP、响应大小和内容类型。
3. 系统拒绝 localhost、环回地址、链路本地地址、私网地址、云元数据地址和不安全重定向。
4. 系统抓取公开页面，提取正文、标题、作者、发布时间、规范 URL 和抓取时间。
5. JavaScript 强依赖、登录墙或反爬导致无法获得正文时，任务失败并给出明确原因。
6. 后续处理与普通文档一致。
7. 首期只支持手动刷新，不递归抓取链接，不定时同步。

### 8.3 AI 问答

1. 用户选择一个或多个有权限的知识空间并输入问题。
2. 系统加载当前会话短期上下文和允许使用的用户长期记忆。
3. 系统进行语言识别、指代消解、查询规范化和必要的问题改写。
4. Agent 判断问题复杂度、关系性、是否需要拆分和是否需要澄清。
5. Agent 根据路由调用混合检索、图谱查询或两者。
6. 系统对候选证据去重、RRF 融合、Reranker 重排并计算证据充分性。
7. 证据不足时最多重写并重试检索一次；仍不足则追问或拒答。
8. 系统使用证据生成答案，按句附加引用，并通过 SSE 流式返回。
9. 用户可打开引用，定位到原文件、页码、工作表、幻灯片或正文位置。
10. 系统记录 Trace、检索候选、模型使用、耗时、Token 和反馈。
11. 会话进入滑动窗口与摘要流程；长期记忆仅在满足写入规则时更新。

### 8.4 语音问答

1. 用户按住说话或点击开始录音。
2. 浏览器通过 WebSocket 上传音频分片。
3. 云端 ASR 返回中间字幕和最终文本。
4. 最终文本进入与文本问答相同的 Agent 流程。
5. 文本答案通过 SSE 输出。
6. 服务端按完整句切分文本并提交流式 TTS。
7. TTS 音频通过 WebSocket 返回，包含 request ID、sequence 和时间戳。
8. 前端按序缓冲和播放，文字显示领先于对应音频。
9. 用户点击停止时，系统取消 Agent 后续步骤、LLM 流、TTS 队列和未播放音频。
10. ASR 失败时可改用文字输入；TTS 失败不影响文字答案。

#### 9.3.1 文档身份、版本与去重

- 每个逻辑文档拥有稳定的 `document_id`，且只属于一个知识空间。
- “创建新文档”和“上传新版本”是两个明确的界面动作和 API，系统不根据文件名自动猜测。
- 创建新文档时，同一知识空间内若已有内容哈希相同的有效文档，系统阻止重复创建并返回现有文档，不创建新版本。
- 跨知识空间上传相同内容时，必须创建独立的逻辑文档、版本、ACL、块和索引记录。底层 MinIO 二进制可按内容寻址去重，但引用计数、授权和删除相互隔离，任何用户都不能感知其他空间存在相同文件。
- 上传新版本时必须指定现有 `document_id`。内容哈希与当前版本相同时记为无变化，不创建新版本；与历史版本相同时引导用户执行“恢复历史版本”。
- 单页 URL 首次导入创建逻辑文档；对该文档执行“刷新 URL”时，内容变化才创建新版本，不变化仅更新最后检查时间。
- 文件名变化不创建新逻辑文档或新版本；文件名和标题属于可独立修改的元数据。
- 解析产物只有在源哈希、解析器版本、OCR 策略和切分策略均相同时才允许内部复用；检索索引和图谱证据始终按空间和文档重新绑定。

### 9.4 解析与切分

- 解析器采用适配器模式，每种格式输出统一的规范化文档结构。
- PDF 保留页码、块坐标、表格和阅读顺序。
- DOCX/DOC 保留标题层级、段落、列表、表格和图片锚点。
- PPTX/PPT 保留幻灯片序号、标题、正文、备注和图片。
- XLSX/XLS/CSV 保留工作表名、表头、行列范围和合并单元格语义。
- JSON 支持对象路径和数组记录切分，避免把整份大 JSON 作为一个块。
- TXT/MD 执行编码检测；Markdown 保留标题层级、代码块和表格。
- 旧版 Office 文件通过隔离的无界面转换服务生成中间格式。
- 扫描 PDF OCR 默认自动启用：页面可提取文本低于版本化文本密度阈值时触发 OCR。
- DOC/DOCX、PPT/PPTX 中的关键图片 OCR 默认启用，可按空间关闭；旧版格式先转换再识别。默认跳过小于 300×100 像素、纯装饰或重复背景图片，判断规则和阈值随解析策略版本记录。
- 切分优先使用标题、段落、列表、表格和页面结构，再受 Token 上限约束。
- 每个子块关联父块、前后块、文档版本、位置和 ACL 快照。
- 表格块同时生成原始结构表示和适合检索的文本表示。
- OCR 结果记录引擎、语言、置信度、页码和坐标。
- 解析和切分必须可重复，参数与解析器版本写入文档版本记录。

### 9.6 Agentic RAG

LangGraph 状态图至少包含：

1. 请求校验与权限范围。
2. 会话上下文和记忆加载。
3. 查询理解与改写。
4. 复杂度与意图路由。
5. 混合检索工具。
6. 图谱查询工具。
7. 证据合并与去重。
8. Reranker。
9. 证据充分性判断。
10. 回答、澄清或拒答。
11. 引用校验。
12. 流式输出。
13. Trace、反馈关联和记忆写入判断。

Agent 约束：

- 只允许调用白名单只读工具。
- 不允许公网搜索、任意 HTTP 请求和业务写操作。
- Neo4j 查询使用受限模板或经过语法与资源校验的只读查询。
- 设置最大步骤数、工具调用数、总超时、单工具超时、Token 和上下文预算。
- 同一个问题最多执行一次自动检索重写。
- 任何重写和扩展都不能突破原始 ACL。
- 文档内容作为不可信数据，不得覆盖系统指令。
- 检索证据不足时不得用模型常识冒充企业事实。
- 外部模型故障时可降级为仅返回排序后的检索结果和原文摘要。

#### 9.6.1 Deep Agents 技术验证边界

- 生产默认 Agent 始终是本节定义的自定义 LangGraph 状态图。Deep Agents 不属于首期核心运行依赖。
- 第二阶段允许在功能开关后增加“复杂跨文档研究模式”技术验证，仅面向关系型、多跳型和需要问题分解的内部知识问题。
- Deep Agents 运行在 Python AI 服务内并复用 LangGraph 运行时、平台模型适配层、数据出网审计和 Langfuse Trace；不得单独持有或绕过平台模型凭证。
- Deep Agents 只暴露混合检索、图谱查询、证据读取和只读记忆工具；不暴露文件写入、Shell、代码执行、任意 MCP、外部网络或业务写操作。
- 主 Agent 与所有子 Agent 使用同一个不可扩大的 ACL 快照；子 Agent 不能自行选择新知识空间或改变数据出网策略。
- 允许验证任务规划和最多 2 个只读子 Agent；总步骤、总超时、Token、模型成本和取消信号由外层 LangGraph/NestJS 统一限制。
- 实验使用独立资源池，最多并发 5 路，不能挤占默认问答容量。
- 子 Agent 的中间结论不能直接进入答案，必须经过统一证据去重、Reranker、证据充分性和引用校验。
- Deep Agents 运行与默认 LangGraph 运行使用相同问题、知识范围和模型进行 A/B 评测，并在 Langfuse 中使用独立策略版本。
- 正式引入门槛：复杂问题正确率相对默认方案提升至少 5 个百分点、引用准确率不低于 0.95、ACL 越权为零、研究模式完成延迟 P95 不超过 30 秒、平均模型用量不超过默认方案 3 倍、取消与 Trace 完整率为 100%。
- 上述指标是“是否采用 Deep Agents”的实验门槛，不是产品发布 SLO 或首期完成门槛。
- 任一门槛未通过时保留实验代码和报告，但功能开关保持关闭，不向普通用户开放，不进入默认发布路径，也不加入生产 Docker Compose 运行依赖；这不阻塞其他阶段验收或产品上线。

### 9.9 短期与长期记忆

短期记忆：

- Redis 保存最近 N 轮滑动窗口和会话摘要。

- 以 user ID 和 session ID 隔离。

- 用户可以查看、编辑、单条删除、全部清除或关闭长期记忆。
- 被删除的长期记忆不能从 Trace、缓存或摘要重新写回。

组织记忆：

- 仅管理员可创建、审核、发布、更新和撤回。
- 不从多用户会话自动沉淀为组织事实。
- 记录来源、适用空间、版本、有效期和发布人。
- 检索组织记忆时遵循空间权限。

## 10. 文档生命周期与状态机

文档生命周期使用两个独立状态字段，禁止把处理进度与在线可见性混在一个枚举中。

### 10.1 处理状态 `processing_status`

1. `UPLOADING`：原文件上传中。
2. `SECURITY_CHECKING`：格式、签名、病毒和资源风险检查。
3. `PARSING`：正文和结构提取。
4. `OCR_PROCESSING`：扫描页或关键图片识别。
5. `CHUNKING`：规范化与结构感知切分。
6. `INDEXING_VECTOR`：空间允许 Embedding 时生成并写入向量；禁止时跳过并记录 `vector_index_status=SKIPPED_POLICY`。
7. `INDEXING_LEXICAL`：写入关键词索引；Elasticsearch 词法索引始终是必需索引。
8. `SEARCHABLE`：词法索引已发布，且向量索引状态为 `PUBLISHED` 或 `SKIPPED_POLICY`。
9. `GRAPH_EXTRACTING`：文档已经可检索，候选图谱在后台生成。
10. `READY`：必需检索索引已完成，候选图谱处理已完成或被空间策略跳过。
11. `PARTIAL_FAILED`：必需索引已发布，但 OCR 部分页或图谱等非阻塞步骤失败，文档仍可检索；失败能力必须在界面明确标记。
12. `FAILED`：必需索引尚未发布，当前版本不可作为在线版本。

### 10.2 可用状态 `availability_status`

1. `DRAFT`：尚无可在线服务的版本。
2. `ACTIVE`：当前发布版本可按 ACL 在线浏览、检索和问答。处理状态可以是 `SEARCHABLE`、`GRAPH_EXTRACTING`、`READY` 或 `PARTIAL_FAILED`。
3. `DISABLED`：人工停用，数据和索引保留，但任何在线入口不可见，可直接恢复为 `ACTIVE`。
4. `SOFT_DELETED`：已进入删除保留期，在线索引和缓存已移除，源文件与解析产物最多保留 30 天，可发起恢复重建。
5. `PURGING`：正在物理清理所有非审计数据，不可恢复。
6. `PURGED`：物理清理完成，仅保留法规和安全所需的最小审计记录。

状态要求：

- 新逻辑文档初始为 `DRAFT`；版本达到 `SEARCHABLE` 后通过原子发布切换为 `ACTIVE`。
- `GRAPH_EXTRACTING` 不降低在线可用性，图谱能力按实际完成状态展示。
- 图谱抽取使用独立 `graph_status`：`PENDING`、`EXTRACTING`、`PENDING_REVIEW`、`COMPLETED`、`PARTIAL_FAILED` 或 `SKIPPED_POLICY`。空间关闭 KB-007 图谱开关时，文档在达到 `SEARCHABLE` 后直接进入 `READY` 并记录 `SKIPPED_POLICY`。
- `PARTIAL_FAILED` 只允许表示非阻塞步骤失败。Elasticsearch 失败时必须为 `FAILED`；空间启用 Embedding 时 pgvector 失败也必须为 `FAILED`；空间禁止 Embedding 时跳过向量不是失败。
- 失败任务从失败节点重试，不重复生成已完成产物。
- 新版本未达到 `SEARCHABLE` 前，旧发布版本和文档 `ACTIVE` 状态保持不变。
- Elasticsearch 与按策略启用的 pgvector 索引通过版本别名或发布标记原子切换。BM25-only 空间只切换 Elasticsearch。
- 停用与删除只改变逻辑文档可用状态，不重写历史版本的处理状态。
- 所有状态转换幂等且可审计。管理员可查看每个阶段的开始时间、结束时间、尝试次数、错误码和关联请求 ID。

## 11. 删除、撤权与一致性

系统不依赖跨数据库分布式事务，采用 PostgreSQL 业务状态、Outbox 事件、Redis Streams 消费组和补偿任务保证最终一致。

软删除流程：

1. PostgreSQL 将逻辑文档设为 `SOFT_DELETED`，记录 `deleted_at` 和 `purge_after`。
2. 权限版本递增并广播缓存失效事件。
3. 立即移除 pgvector 和 Elasticsearch 在线索引记录。
4. Neo4j 将对应证据和仅由该证据支持的自动关系标记为不可用，但在保留期内保留恢复所需记录。
5. 删除 Redis 缓存、短期会话引用和相关任务临时数据。
6. MinIO 原文件和规范化解析产物在 30 天保留期内继续保存，但任何用户下载、预览、检索或 Agent 工具均不可访问。
7. 生成软删除报告。任一步骤失败都进入重试和死信流，`SOFT_DELETED` 在线不可见状态不得回滚。

恢复流程：

1. 仅允许在 `purge_after` 前由系统管理员或空间 `manage` 成员发起。
2. 文档先回到 `DRAFT`，从保留的解析产物重建必需的关键词索引、按当前空间策略启用的向量索引，以及按图谱策略启用的证据绑定。
3. Elasticsearch 和按策略启用的向量索引重新达到 `SEARCHABLE` 后，文档才能原子切换为 `ACTIVE`。

物理清理流程：

1. 保留期到期后将状态设为 `PURGING`，禁止恢复。
2. 删除 MinIO 原文件和解析产物、文档版本、块、向量、关键词记录、Neo4j 证据和非必要业务记录。
3. 内容寻址对象仅在跨空间引用计数为零时物理删除，不能影响其他空间的独立文档。
4. 清理完成后状态设为 `PURGED`，只保留文档 ID、空间 ID、操作者、时间、删除原因和清理结果等最小审计信息。
5. 失败步骤进入重试和死信流，生成跨存储物理清理报告。

一致性修复：

- 定时比对文档版本、块、向量、Elasticsearch 记录和 Neo4j 证据引用。
- 缺失索引可从规范化解析产物重建。
- 孤立向量、关键词记录和图谱证据进入隔离并清理。
- 修复过程不得让已删除或无权限内容短暂可见。

## 12. 总体技术架构

### 12.1 React 前端

- 用户端与管理端共用设计系统和权限路由。
- 负责文档上传、任务状态、问答、引用预览和语音播放。
- 使用 `@ai-sdk/react` 的 `useChat` 和自定义 `ChatTransport` 管理会话消息、发送、流式状态、取消和错误。
- 文本问答使用 AI SDK `UIMessage` 与 UI Message Stream；Agent 状态、检索摘要和引用通过类型化自定义 data parts 表达。
- 自定义 Transport 统一附加身份令牌、会话 ID、知识空间 ID、客户端 request ID 和协议版本。
- Vercel AI SDK 仅用于前端交互与流协议，不在浏览器或 NestJS 中直接调用云模型；所有模型调用继续由 Python AI 服务和受控模型适配层执行。
- WebSocket 继续独立处理 ASR/TTS 音频，不强行复用文本聊天 Transport。
- 在项目适配层封装 AI SDK 类型和 Transport，固定依赖版本，并通过契约测试控制升级影响。
- 客户端不保存模型密钥。
- 所有 Markdown 和引用内容经过安全渲染。

### 12.2 NestJS 主后端

负责：

- 本地账号、令牌、部门、角色和 ACL。
- 知识空间、文档元数据、版本、会话、消息、反馈和审计。
- 对外 REST API、SSE 和 WebSocket 网关。
- 将 Python/LangGraph 内部事件转换为 AI SDK UI Message Stream，并校验协议版本和 data part 结构。
- 文件上传签名、下载授权和请求限流。
- 任务记录、Outbox、Redis Streams 发布和状态聚合。
- Python AI 服务、模型配置和数据存储的统一业务边界。

### 12.3 Python AI 服务

负责：

- 多格式解析、OCR、规范化和结构感知切分。
- 按空间策略执行 Embedding 与 pgvector 写入，并始终构建 Elasticsearch 索引。
- RRF、Reranker、上下文组织和引用对齐。
- LangGraph Agent 状态图和只读工具实现。
- Deep Agents 复杂跨文档研究模式实验实现与对比评测。
- Neo4j 候选图谱抽取、实体链接、路径查询和证据回填。
- Mem0 长期记忆。
- ASR/TTS 供应商适配。
- 离线评测和质量指标计算。

### 12.4 内部通信

- NestJS 与 Python 使用内部 HTTP/OpenAPI 处理同步控制请求。
- 长耗时导入、索引、图谱和评测任务使用 Redis Streams 消费组。
- PostgreSQL 任务表是业务状态真相源，Redis 不作为唯一持久状态。
- 事件必须包含 event ID、task ID、resource ID、version、attempt 和 trace ID。
- 消费者按 event ID 幂等处理。
- 失败任务按指数退避重试，超过阈值进入死信流。

### 12.5 数据存储

| 组件 | 责任 |
|---|---|
| PostgreSQL | 用户、权限、知识空间、文档、版本、任务、会话、反馈、策略和审计 |
| pgvector | 文档块 Embedding 与向量召回 |
| Elasticsearch | BM25、中文分词、字段过滤和高亮 |
| Neo4j | 已审核和候选实体、关系、路径与证据关联 |
| Redis | Redis Streams、缓存、限流、短期记忆和会话摘要 |
| MinIO | 原文件、规范化解析产物、OCR 产物和可重建中间数据 |
| Langfuse | AI Trace、Prompt、模型、Token、反馈和评测 |
| Prometheus/Grafana | 系统指标、看板和告警 |

### 12.6 云端模型适配层

- 使用统一接口屏蔽供应商差异。
- 系统管理员按 LLM、Embedding、Reranker、ASR、TTS 能力配置平台级主供应商和可选降级供应商，不要求同一家提供所有能力。
- 知识空间只能控制能力开关、脱敏、配额和保留策略，首期不能选择与平台不同的供应商或模型。
- Embedding 模型、维度和距离度量属于平台级索引代次；切换 Embedding 必须创建新索引代次、完成全量重建和评测后原子发布。
- 请求记录供应商、模型、用途、数据分类、Token、耗时和结果状态。
- 支持超时、有限重试、熔断、并发限制和配额。
- 供应商切换必须通过评测集和数据合规检查。
- 知识空间可禁止任何云模型调用；此时只提供本地可完成的文档浏览和关键词检索能力。
- 多空间请求严格执行第 9.2 节的策略交集，禁止按供应商方便性静默排除受限空间。

## 13. 核心数据模型

### 13.1 PostgreSQL 核心实体

- `User`：账号、部门、状态、系统角色和安全信息。
- `Department`、`Group`：组织与授权集合。
- `KnowledgeSpace`：空间配置、默认策略和数据出网策略。
- `SpaceMember`：主体、权限级别和有效期。
- `Document`：单一知识空间内的逻辑文档、当前发布版本和 `availability_status`。
- `DocumentVersion`：所属逻辑文档、源文件、内容哈希、解析器版本、索引版本和 `processing_status`。
- `ContentObject`：按内容哈希寻址的内部 MinIO 对象和跨空间引用计数，不承载用户可见权限。
- `Chunk`：父子关系、位置、文本、结构类型、ACL 与内容哈希。
- `ImportTask`、`TaskAttempt`：任务阶段、进度、错误和重试。
- `Conversation`、`Message`：会话、消息、生成状态和策略版本。
- `Citation`：消息句段与块、页码、区域的关联。
- `Feedback`：用户反馈、分类、说明和处理状态。
- `MemoryPolicy`：空间和用户记忆策略。
- `ModelProvider`、`ModelConfig`：能力、模型、凭证引用、限额和状态。
- `RetrievalPolicy`、`PromptVersion`：版本化策略。
- `EvaluationDataset`、`EvaluationCase`、`EvaluationRun`、`EvaluationResult`。
- `AuditLog`、`EgressLog`：操作和数据出网审计。
- `OutboxEvent`：跨存储和异步任务事件。

### 13.2 Neo4j 核心结构

- `Entity` 节点：规范名称、别名、类型、语言和发布状态。
- `Document`、`Chunk` 证据引用节点或稳定外部 ID。
- 业务关系：关系类型、置信度、抽取版本、审核状态、有效时间。
- `SUPPORTED_BY`：实体或关系到来源块。
- `SAME_AS`、`ALIAS_OF`：实体消歧和合并历史。
- 草稿与发布状态不得仅依靠前端隐藏。

## 14. API 与流式协议边界

对外 API 分组：

- `/auth`：登录、刷新、退出和个人信息。
- `/users`、`/departments`、`/groups`：本地账号与组织。
- `/spaces`：知识空间和权限。
- `/documents`、`/imports`：上传、URL 导入、版本、状态、预览和删除。
- `/search`：授权范围内的文档搜索。
- `/conversations`、`/messages`：会话历史。
- `POST /chat/stream`：兼容 AI SDK UI Message Stream 的 SSE 文本问答。
- `POST /chat/:requestId/cancel`：幂等取消 Agent、模型流和待生成 TTS。
- `GET /citations/:citationId/resolve`：按当前用户、当前 ACL 和当前文档状态重新鉴权后返回引用详情与不透明预览会话。
- `/voice`：WebSocket ASR/TTS 会话。
- `/graph`：实体、关系、路径和治理。
- `/memories`：个人记忆与组织记忆。
- `/evaluations`：评测集、运行和结果。
- `/admin/models`、`/admin/policies`、`/admin/observability`。
- `/audit`：管理员审计查询。

文本聊天协议：

- 外部协议采用 AI SDK UI Message Stream；React `useChat` 通过自定义 `ChatTransport` 调用 NestJS，不直接访问 Python 服务。
- 标准文本使用 AI SDK 文本 message parts；通用来源优先使用 source parts，企业文档页码、坐标和工作表使用类型化 `data-citation`。
- 应用自定义 data parts 至少包含 `data-request-metadata`、`data-agent-status`、`data-retrieval-summary`、`data-citation` 和 `data-error-detail`。
- 浏览器端 `data-citation` 仅包含不透明 `citation_id` 和展示所需的文档标题、位置、片段摘要；不得包含用户、部门、用户组、权限规则、内部 ACL 快照、MinIO 对象键或可长期使用的下载地址。
- 打开引用时前端只提交 `citation_id`。NestJS 必须根据当前登录用户、最新 ACL、文档可用状态和版本重新鉴权；生成答案时的 ACL 快照和客户端字段都不能作为当前授权依据。
- 引用预览内容经 NestJS 鉴权代理读取；不透明预览会话最长 5 秒且每次页面读取重新校验，不向浏览器暴露可绕过撤权的 MinIO 直链。
- 引用生成后发生撤权、停用、软删除或版本失效时，解析接口返回无权访问或已失效，不得继续展示缓存正文。
- `data-request-metadata` 包含 request ID、trace ID、策略版本和递增 sequence，不包含模型思维链。
- NestJS 内部仍使用 `message.started`、`agent.status`、`retrieval.completed`、`content.delta`、`citation`、`message.completed`、`message.cancelled` 和 `error` 事件，但这些不是浏览器直接依赖的传输格式。
- 服务端设置明确的聊天协议版本；前端发送支持版本，版本不兼容时在开始生成前失败。
- AI SDK 依赖升级必须通过保存的流协议契约样例、取消、错误和引用回放测试。
- 首期不自动恢复已断开的流，也不自动重新执行请求；客户端重新加载后读取已持久化的最终状态或明确的中断状态。

WebSocket 音频事件至少包括：

- `session.start`
- `audio.input`
- `asr.partial`
- `asr.final`
- `tts.audio`
- `tts.completed`
- `session.cancel`
- `session.error`

所有文本和音频流式事件都可关联 request ID。客户端断线重连不得重复执行已取消或已完成的请求。

## 15. 安全、隐私与合规

### 15.1 文件与解析安全

- 扩展名、MIME 和文件签名三重校验。
- 上传大小、页数、解压大小、嵌套层数和解析时长受限。
- 病毒扫描和压缩炸弹检测。
- 宏和嵌入对象默认不执行。
- LibreOffice、OCR 和解析器运行在受资源限制的隔离进程。
- 原始文件名不能直接作为服务器路径。
- 预览和下载使用短时授权地址。

### 15.2 URL 安全

- 仅允许 HTTP/HTTPS。
- 每次 DNS 解析和重定向后都检查目标地址。
- 拒绝环回、私网、链路本地、保留地址和云元数据服务。
- 限制重定向次数、响应大小、下载时间和内容类型。
- 不携带服务器环境中的 Cookie、Authorization 或代理凭证。
- 抓取内容按不可信输入处理。

### 15.3 云模型数据出网

- 知识空间设置是否允许调用云模型。
- 只发送完成当前任务所需的最小文本或音频。
- 可配置 PII、密钥、账号和身份信息脱敏。
- 敏感空间可禁止 LLM、Embedding、Reranker、ASR 或 TTS 中的任一能力。
- 每次出网记录用户、空间、供应商、模型、用途、数据分类、Token 和结果。
- 上线前确认供应商的数据保留、训练使用、地域和删除政策。
- 日志和 Trace 默认不记录完整敏感正文。

### 15.4 应用安全

- 密码使用 Argon2id 或同等级强哈希。
- API 使用 TLS；生产通过反向代理终止 TLS。
- 密钥使用专门的加密密钥保护，禁止写入 Git。
- 防止越权、CSRF、XSS、SQL/Cypher 注入和路径穿越。
- API、登录、问答、上传和模型调用分别限流。
- 文档 Prompt Injection 不能改变系统指令或工具权限。
- Agent 只读且工具白名单固定。
- 审计日志不可由普通管理员随意修改。

### 15.5 默认数据保留

- 软删除原文件和解析产物默认保留 30 天，之后执行物理删除。
- 操作审计和数据出网审计默认保留 365 天。
- 任务运行记录、错误详情和系统诊断日志默认保留 180 天。
- Langfuse Trace 元数据默认保留 90 天；包含授权调试正文的采样 Trace 默认保留 7 天。
- 原始录音默认不持久化；ASR 最终文本按普通会话消息管理。
- 用户主动清除长期记忆后立即在线不可见，并在 24 小时内完成物理清理。
- 保留期可由管理员调整，但缩短审计保留期需要二次确认并记录原因。

## 16. 可观测性

### 16.1 系统可观测

- 使用 OpenTelemetry 贯穿 React 请求、NestJS、Python、任务、存储和模型调用。
- correlation ID、trace ID、request ID 和 task ID 可相互定位。
- Prometheus 采集请求量、延迟、错误率、队列深度、任务耗时、资源和存储指标。
- Grafana 提供系统总览、导入、问答、模型、语音和存储看板。
- 日志结构化输出，敏感字段脱敏。

### 16.2 AI 可观测

Langfuse 记录：

- LangGraph 节点和工具调用。
- 查询改写、检索候选 ID、排名和策略版本。
- Prompt 版本、模型、Token、延迟、错误和重试。
- 引用集合、证据充分性和回答状态。
- 用户反馈、离线评测和版本对比。

生产环境默认只记录证据 ID 和必要摘要。完整 Prompt 或正文采样需要管理员授权、用途说明和保留期限。

## 17. 非功能需求

### 17.1 容量

- 原文件总量不超过 500 GB。
- 活跃文档不超过 10 万份。
- 注册用户不超过 1,000。
- 峰值同时进行的问答请求为 50。
- 导入任务与在线问答使用独立并发和资源配额，批量导入不得拖垮在线问答。

### 17.2 性能

在约定生产硬件和云模型满足供应商 SLA 的条件下：

- 混合检索 P95 不超过 1.5 秒，不含最终 LLM 生成。
- 文本问答端到端首字 P95 不超过 6 秒。
- SSE 心跳间隔不超过 15 秒。
- ASR 中间结果 P95 不超过 1 秒。
- 第一完整句形成后，TTS 首音频 P95 不超过 2.5 秒。
- 常规管理列表 P95 不超过 2 秒。
- 文档撤权后 5 秒内所有在线缓存路径失效。

### 17.3 可用性与恢复

- 单机 Docker Compose 月度目标可用性为 99.5%，不含计划维护和云供应商故障。
- RPO 不超过 24 小时。
- RTO 不超过 8 小时。
- 每日自动备份 PostgreSQL、MinIO、Neo4j 和关键配置。
- 至少每季度执行一次恢复演练。
- Redis 丢失不得造成永久业务状态丢失。
- Elasticsearch 和向量索引可从规范化解析产物重建。

### 17.4 兼容与易用性

- 支持 Chrome 和 Edge 最近两个主要版本。
- 管理界面和用户界面中文优先。
- 支持中文、英文和中英混合文档检索。
- 关键流程支持键盘操作、清晰焦点和可读错误提示。
- 导入、问答和语音的长任务都有可见进度和取消能力。

## 18. 质量指标与验收

### 18.1 评测数据

- 不少于 200 个经过人工确认的真实问题。
- 覆盖事实型、关键词型、语义型、关系型、多跳型、表格型、跨文档型、不可回答型和中英混合型问题。
- 每个问题有标准证据；可回答问题有答案要点，不可回答问题有拒答预期。
- 训练、调参和最终验收集合分离，避免对验收集过拟合。

### 18.2 RAG 质量门槛

| 指标 | 首发门槛 |
|---|---:|
| Recall@10 | ≥ 0.85 |
| nDCG@10 | ≥ 0.75 |
| 引用准确率 | ≥ 0.95 |
| 答案忠实度 | ≥ 0.90 |
| 不可回答处理正确率 | ≥ 0.85 |

指标必须按问题类型和知识空间同时展示，不能只报告总体平均值。

指标口径：

- `Recall@10`：标准证据集合中至少一个有效证据出现在前 10 个候选的案例比例。
- `nDCG@10`：按人工标注的证据相关性等级计算前 10 个候选的归一化折损累计增益。
- 引用准确率：抽样答案中，被引用原文实际支持对应陈述的引用数量占全部引用数量的比例。
- 答案忠实度：答案中可由检索证据直接支持的事实陈述占全部事实陈述的比例。
- 不可回答处理正确率：不可回答案例中，系统正确拒答或提出必要澄清问题的比例。
- 引用准确率和答案忠实度可使用固定评审模型初评，但每次发布验收必须由人工复核全部失败案例和至少 10% 的通过案例。
- 性能与质量门槛需连续三次完整运行均通过，才允许发布为默认策略。

### 18.3 功能验收

- 每种受支持格式至少包含正常、损坏、超限、中文、英文和边界样例。
- 扫描 PDF 能引用到正确页码和 OCR 区域。
- 表格问题能引用到正确工作表和行列区域。
- 旧版本在新版本索引完成前保持服务，新版本发布后不混用块。
- 删除或撤权后 pgvector、Elasticsearch、Neo4j、Redis 和预览均不可命中。
- 图谱关系可回到来源片段，未审核候选不会作为组织事实展示。
- 用户可查看和删除个人长期记忆。
- SSE 和 WebSocket 都支持取消、异常和断线处理。

### 18.4 安全验收

- 越权检索和引用测试必须为零通过案例。
- SSRF 测试覆盖私网、环回、DNS 重绑定和重定向绕过。
- 上传测试覆盖伪造扩展名、病毒样本、压缩炸弹和恶意 Office 文件。
- 发布时高危漏洞为零；中危漏洞有明确负责人和修复期限。
- API 密钥、密码、完整敏感正文不得出现在日志和前端响应中。

### 18.5 测试矩阵

- 单元测试：权限、状态机、切分、RRF、记忆门控、引用对齐。
- 集成测试：PostgreSQL、pgvector、Elasticsearch、Neo4j、Redis、MinIO。
- 契约测试：Vercel AI SDK UI Message Stream 与自定义 data parts、React 与 NestJS、NestJS 与 Python、系统与各云模型供应商。
- 端到端测试：登录、授权、导入、更新、问答、引用、反馈、删除、图谱和语音。
- 质量回归：200+ 问题评测集。
- Agent 对比测试：默认 LangGraph 与 Deep Agents 实验模式在同一复杂问题子集上的质量、引用、ACL、延迟、Token、取消和 Trace。
- 压力测试：50 路问答、并行导入和大文件处理。
- 故障注入：模型超时、Redis 重启、索引写入失败、磁盘不足和 WebSocket 断线。
- 恢复测试：从备份恢复、索引重建和删除补偿。

## 20. 错误处理与降级

| 场景 | 用户行为 | 系统处理 |
|---|---|---|
| 文件不支持或损坏 | 显示具体原因，不进入索引 | 保留最小任务记录，可重新上传 |
| OCR 局部失败 | 文档可进入部分可用状态 | 标记失败页，允许仅重试 OCR |
| 已启用的向量索引失败 | 旧版本继续服务 | 重试失败步骤，不发布新版本；策略禁用向量时不视为失败 |
| Elasticsearch 失败 | 旧版本继续服务 | 进入重试或死信，不发布半成品 |
| 图谱抽取失败 | 文档仍可检索问答 | 标记部分失败，允许稍后重试 |
| 云 LLM 超时 | 提示生成失败或返回检索结果 | 有限重试、熔断、记录 Trace |
| Reranker 失败 | 继续使用 RRF 排名 | 标记降级，不阻断问答 |
| 证据不足 | 追问或拒答 | 不用模型常识补齐企业事实 |
| AI SDK 流协议版本不兼容 | 不开始生成，提示刷新或升级客户端 | NestJS 在调用 Python 前校验版本并返回结构化错误 |
| SSE 断线 | 保留已生成文本 | 请求可取消，不重复写入最终消息 |
| Deep Agents 实验模式超时 | 停止子 Agent 并保留可审计失败状态 | 不自动重复产生费用，提示改用默认问答模式 |
| TTS 失败 | 保留文字答案 | 停止音频并允许重试播报 |
| Redis 重启 | 短期缓存和队列可恢复 | 永久业务状态以 PostgreSQL 为准 |
| 存储空间不足 | 暂停新导入并告警 | 不破坏现有索引，提供清理建议 |

所有用户可见错误包含：

- 简明错误说明。
- 当前任务状态。
- 用户可以执行的下一步。
- 可提供给管理员的关联请求 ID。

## 22. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 旧版 Office 与复杂排版解析失真 | 引用错误、检索缺失 | 隔离转换、格式回归集、原文预览、失败页提示 |
| OCR 对低质量扫描件识别差 | 错误实体和答案 | 置信度门槛、页面重试、人工替换文本、评测集覆盖 |
| 云模型处理敏感数据 | 合规风险 | 空间级禁用、最小片段、脱敏、出网审计、供应商审查 |
| 多存储数据不一致 | 删除后仍可召回 | Outbox、幂等事件、补偿任务、一致性扫描 |
| 图谱抽取幻觉 | 错误多跳推理 | 候选/发布隔离、证据绑定、人工审核、置信度与版本 |
| Agent 循环或成本失控 | 延迟和费用异常 | 最大步骤、工具次数、超时、Token、熔断和配额 |
| Deep Agents 规划或子 Agent 开销超过收益 | 复杂度、延迟和成本上升 | 默认关闭、只读工具、固定 ACL、A/B 门槛，未通过不进入生产路径 |
| Vercel AI SDK 升级改变流协议 | 聊天 UI 与 NestJS 不兼容 | 适配层封装、固定版本、协议版本协商和契约回放测试 |
| 文档 Prompt Injection | 越权或错误行为 | 文档不可信、只读白名单工具、指令隔离、输出引用 |
| 单机 Compose 故障 | 服务中断 | 99.5% 边界声明、每日备份、恢复演练、后续 K8s 路线 |
| 模型供应商锁定 | 成本和迁移风险 | 统一适配层、模型独立配置、契约测试和评测准入 |
| 评测集偏差 | 指标好但真实体验差 | 真实问题分层、保留验收集、线上反馈人工筛选 |

## 24. 完成定义

本产品达到“首期真实可用”的条件是：

1. 企业成员能够在授权范围内导入、浏览、检索和问答全部约定格式。
2. React 通过 Vercel AI SDK UI 接收 NestJS 的兼容流，关键事实有可打开的类型化原文引用，取消和错误状态一致。
3. 复杂关系问题可由 Agent 动态调用图谱和混合检索，且每条图谱结论有文档证据。
4. 证据不足时系统能够澄清或拒答。
5. 文档更新、撤权和删除能够跨 pgvector、Elasticsearch、Neo4j、Redis 和 MinIO 一致生效。
6. 用户私有记忆可控，组织记忆经管理员审核。
7. 半双工语音链路可流式识别、生成、合成、播放和取消。
8. 管理员能够追踪任务、AI Trace、反馈、评测、审计和数据出网。
9. 所有已启用的生产功能达到本文质量、性能、安全和恢复门槛；保持关闭的 Deep Agents 实验不属于产品发布门槛。
10. Docker Compose 部署、备份、恢复和运维文档完整。
